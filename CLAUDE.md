# da Cecot Food — Identity Card

Custom **static** rebuild of dacecotfood.com (authentic Italian comfort-food pasta bar & street-food kitchen, Edmonton AB). Client project for **Unconventional Group**. This is the dacecot site — NOT a framework app, NOT another caterer.

## What it is / who it's for
- 13-page marketing site for da Cecot Food Inc. Production-ready: full SEO/AIO, structured data, responsive, accessible.
- Pages: home, menu, about/our-story, experiences (hub) + 5 sub-pages (sunday-pasta-classes, pasta-drop-in / pasta-shop, food-drink-experiences, private-events, catering), events, reservations, contact/visit-us, partnerships. (On-disk filenames include `our-story.html`, `pasta-shop.html`, `visit-us.html` — README lists older slugs; trust the generator output.) Plus `404.html`, `sitemap.xml`, `robots.txt`, `favicon.svg`.

## Brand / design system
- cream **#f9f7ef**, brown **#4a1e18**, olive **#374225**, terracotta **#ad5217**, gold **#c4a035**.
- **Playfair Display** (headings) + **Inter** (body). Experiences dropdown nav, FAQ accordions, alternating sections.
- dacecotfood.com itself is the quality bar referenced for other UGroup builds.

## Tech stack
- Plain static HTML/CSS/vanilla JS, NO runtime framework. Pages are **generated by a Node build script** — do NOT hand-edit the `.html` files (overwritten on build).
  - `.claude/build.js` — head/SEO, nav (Experiences dropdown), footer, schema helpers + the 7 core pages.
  - `.claude/build-experiences.js` — the 6 Experiences pages + final writer (sitemap, robots, 404, favicon, CSS minification).
- `css/styles.css` (source) → `css/styles.min.css` (generated, linked by pages). `js/main.js` = mobile nav, dropdown, FAQ accordion, scroll reveal, back-to-top, forms.

## Build
- `node .claude/build.js` regenerates all 13 pages + 404 + sitemap + robots + favicon + styles.min.css. **Edit the generator, then re-run** — editing `.html` directly gets overwritten.

## SEO / AIO
- Unique title/meta/canonical/OG/Twitter per page; one `<h1>` each; semantic structure; alt text + lazy/async images.
- JSON-LD: Restaurant (home), Restaurant+LocalBusiness (contact), Event (classes/drop-in), Service (experiences/private-events/catering), FAQPage, BreadcrumbList. "What is…" answer paragraphs + FAQ accordions for AI Overviews.

## Deploy

## Local preview
- `node .claude/static-server.js` → http://localhost:4321.

## PLACEHOLDERS / gotchas
- **Images** sourced from the client's Wix CDN (live-site real photos). The client's Google Drive originals folder is **private/download-locked (403)** — can't auto-import; see `DRIVE-PHOTOS.md` for filename/ID inventory + the access issue.
- The **contact form** (visit-us) is a front-end mockup (FormSubmit, no backend).
- **Reservations** route to the client's **Wix reservation page** — a live "Book a Table" button (opens Wix in a new tab). URL lives in `RESERVATION_BOOKING_URL` near the top of `.claude/build.js` (currently `https://dacecotfood.wixsite.com/my-site` — the bare `dacecotfood.wixsite.com` subdomain 404s; the published page is at the `/my-site` path). If that constant is ever blanked, the Reserve page falls back to a clearly-marked "booking launching shortly" pending state and steers guests to phone/email. (We previously scaffolded Square Appointments for this; that was dropped in favour of Wix on 2026-06-27.)
- **Pasta classes still use Square**, intentionally: Sunday Pasta Classes book + pay $95/guest via a Square checkout form (`square.link/u/mTkWSnl5`) on the Experiences page, and the Thursday drop-in uses a free FormSubmit reservation form. Only table reservations moved to Wix.
- **Menu page** surfaces PDFs from `/menus` (`menuCards` array in `build.js`): Dinner, Lunch, Drink, Dessert are live; Liquor & Kids are "coming soon". Lunch = `lunch-menu.pdf` (Erika's lunch menu), Dessert = `dessert-menu.pdf` (drink + dessert menu). Drop a PDF in `/menus`, flip `ready:true`, re-run the build.
